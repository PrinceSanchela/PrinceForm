Security Guidelines

Last Updated: July 13, 2026

At Prince Form, security is built directly into our core architecture. We implement strict defense mechanisms to safeguard creator dashboards and responder submissions from unauthorized access.

1. Password Protection (Salted Cryptographic Hashing)

Passwords are never stored in plain text. When you register or update your password:

  • The system applies industry-standard cryptographic salting and key derivation algorithms.
  • This ensures that your private credentials remain encrypted and secure, even in the event of database access.
  • We enforce complexity validation checks on both the client and server during password creation.

2. Brute-Force Rate Limiting

To defend against automated credential cracking attempts, our authentication system actively monitors traffic patterns:

  • Repeated consecutive failed login attempts on an account trigger an automated temporary lockout.
  • All subsequent login queries for the account are immediately rejected during the lockout window, returning the time remaining before release.
  • Any successful login resets the attempt tracker to 0.

3. Secure Session Architecture

Active user sessions are managed securely on the server side:

  • Authentication relies on cryptographically random, server-validated session identifiers.
  • Session cookies are stored on the browser using strict `HttpOnly` and `SameSite` flags. This guards your credentials against cross-site scripting (XSS) and cross-site request forgery (CSRF) attempts.
  • Logging out instantly invalidates and purges the session record from our database.

4. Database Access Isolation

All database queries are strictly authenticated and isolated:

  • Every request to fetch, edit, or delete forms is verified against the creator's active session.
  • This scopes database calls to the individual owner, making it impossible for accounts to view or manipulate data belonging to other users.

5. Secure Email Recovery

Password recovery utilizes temporary, single-use verification pins dispatched via encrypted SSL/TLS email delivery. These codes are marked as used immediately upon submission and expire shortly after creation to prevent reuse or replay attacks.
